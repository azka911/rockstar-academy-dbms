# Rockstar Academy — Database Management System

A database design project for Rockstar Academy, a sports and performing arts academy for kids and teens in Asia. The project was built through stakeholder interviews with an area manager overseeing 6 academy branches, translating real operational requirements into a fully normalised relational database schema.

## Project Overview

Rockstar Academy manages over 100 classes per week across multiple branches, handling member enrollment, class scheduling, coach assignments, attendance tracking, and progress reporting. This project designs a database system to support those operations efficiently.

## My Role

Business Analyst & Database Designer

- Authored the Business Requirements Document (BRD) covering member management, class scheduling, attendance, progress tracking, financial management, and special support services
- Designed the normalised Oracle schema from UNF through to 3NF, resolving a transitive dependency in the MEMBER table via a dedicated PARENT_INFO table
- Modelled 9 entities across 8 tables plus 3 bridge tables for many-to-many relationships

## Tech Stack

Oracle SQL · ERD Design · Data Normalisation (UNF → 3NF)

## Database Schema

The schema covers 9 entities:

- `MEMBER` — registered students with membership and session data
- `PARENT_INFO` — separated from MEMBER in 3NF to resolve transitive dependency
- `CLASS` — classes categorised by type, age group, skill level, and capacity
- `COACH` — coaches assigned to classes by skill level and gender requirement
- `MANAGER` — managers overseeing one or more academy branches
- `ACADEMY` — individual academy branches
- `EQUIPMENT` — equipment assigned to classes by skill level
- `SPECIAL_SUPPORT` — optional support services for specially-abled members
- `PROGRESS_REPORT` — term-end reports generated by coaches for qualifying members

Bridge tables handle many-to-many relationships between MEMBER↔CLASS, CLASS↔EQUIPMENT, and CLASS↔SPECIAL_SUPPORT.

Key business rules enforced at schema level:
- Members can register for a maximum of 21 classes
- Class capacity capped at 30 students
- Membership type restricted to VIP or Normal
- Coach gender constraints for certain class types
- Progress reports only generated for members meeting minimum attendance (8 sessions/month)
